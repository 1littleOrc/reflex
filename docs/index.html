Reflex

a lightweight responsive flexbox grid with cross browser support, an inline-block fallback and no polyfills

What is Reflex?

Reflex gives you the ability to easily take advantage of flexbox for laying out a grid while having a reliable fallback for older browsers that do not support flexbox. Browser support for flexbox is in a pretty good state right now and is growing all the time. At the time of writing, full global flexbox support is at 81.74% (excluding browsers with outdated or partial support) so I propose that it's acceptable to start taking advantage of flexbox and accept that the inline-block fallback won't support everything. You'll still have a working grid on older browsers (all the way back to ie7), just not all of the cool flexbox stuff that doesn't have an inline-block equivalent such as order

The benefits are:

  • It's lightweight - 8.5KB in it's minified form and 1.6KB gzipped
  • Where flexbox is supported, grid items are all the same height by default
  • Grid items never push each other out of the way (as with floated grids)
  • Good cross browser support

It's intended to be used as:

  • A means of generating a reliable grid with dynamic content e.g. a list of products
  • An addition to your current css framework
  • A set of vendor-prefixed helper classes and LESS mixins to get your flexbox solution off the ground faster

It's not intended to be used as:

  • A framework for full page layout (although you probably could if you wanted to)
  • A replacement for bootstrap or any other css framework

Reflex grid introduction

The reflex-container class sets a section of your page up as display: flex. It's children with a class like reflex-col-6 are also set as display: flex with a fallback of display: inline-block with strict widths which allows them to act as a grid. The idea is that doing this allows us to take advantage of the extra layout features of flexbox for browsers that support it while also providing a working inline-block grid for those that don't.

By default Reflex is set up as a 12 column grid but it can be modified to support more or less columns as required

As a consequence of the grid container and the grid elements being set to display: flex most of the helper classes can be applied to the grid items themselves which gives you a really flexible grid structure with (for example) column based grids inside row based grids. There are some examples later on to demonstrate some of this.

Here is an example of the bare minimum markup you need to create a grid:

<div class="reflex-container">
    <div class="reflex-col-6">
        <content>
    </div>
    <div class="reflex-col-6">
        <content>
    </div>
</div>

Browser support

Reflex should fall back to an inline-block layout for browsers that do not support flexbox explicitly. Reflex uses some css hacks to help the inline-block grid work on older browsers. It's important to note that some of the features of flexbox do not have an equivalent in the inline-block fallback so not everything will work in the fallback, but your grid will remain intact.

If you want to use the grid with breakpoints in ie8 and below you will need to use respond.js which enables media query support in those browsers via JavaScript.

Making the grid wrap

You can add the class reflex-wrap to the container which sets flex-wrap: wrap and forces it to wrap into rows.

In the inline-block fallback your grid items will wrap automatically when they add up to 100% width (12 column units by default)

<div class="reflex-container reflex-wrap">
    <div class="reflex-col-6">
        <content>
    </div>
    <div class="reflex-col-6">
        <content>
    </div>
    <div class="reflex-col-8">
        <content>
    </div>
</div>

Cross axis alignment (align-items)

There are helper classes for easily setting the value of align-items with some extra css for the inline-block fallback where possible. These helper classes can be added to your reflex-container element but also to the reflex-col-x elements inside the container. This allows you to control layout of the entire grid and also individual elements in the grid.

Equal height columns

The use of display: flex gives us equal height divs by default in browsers that support flexbox because the default value of align-items is stretch

There is no reliable fallback for same height inline-block elements so in browsers that do not support flexbox your columns will not be of equal height unless the content happens to be all the same height.

This property has no effect when there is only one line of items in the container so relies on the usage of reflex-wrap on the container

Ordering

These helper classes allow you to easily set order to your grid elements and control the rendered order. By default there are 12 ordering helpers. If you need more then they can be modified

Ordering has no support in the inline-block fallback

.reflex-order-<n>

uses order: n

<div class="reflex-container">
    <div class="reflex-col-4 reflex-order-2">
        <p>I am <strong>first</strong> in the markup but ordered second</p>
    </div>
    <div class="reflex-col-4 reflex-order-3">
        <p>I am <strong>second</strong> in the markup but ordered third</p>
    </div>
    <div class="reflex-col-4 reflex-order-1">
        <p>I am <strong>third</strong> in the markup but ordered first</p>
    </div>
</div>
