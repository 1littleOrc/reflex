Reflex

What is Reflex?

Reflex is a responsive flexbox grid with inline-block legacy support that works on the basis that in the spirit of progressive enhancement it's ok that things look a bit different in older browsers. It gives you the ability to easily take advantage of flexbox for layout with a reasonable feeling of warmth and security that things will still look ok in older browsers.

The fallback to inline-block grid instead of one that uses floats is because it avoids that horrible situation where floats overlap each other and avoids having to use rows to solve that problem. The downside is worse legacy support.

Browser support

Browser support for flexbox is in a pretty good state right now across the major browsers and regardless, Reflex should fall back to an inline-block layout for most browsers that do not support flexbox explicitly. Reflex uses some css hacks to help the inline-block grid work on older browsers.

If you want to use the grid and have it work in ie8 and below you will need to use respond.js which enables media query support in those browsers via JavaScript.

IE7 does not support the box-sizing model which this grid uses. So if you need to support IE7 you will need to avoid using the reflex-padding-* helper classes and instead apply margin to a child element of the grid columns

Reflex grid introduction

The reflex-container class sets a section of your page up as display: flex. It's children with a class like reflex-col-xs-6 are also set as display: flex with a fallback of display: inline-block with strict widths which allows them to act as a grid. The idea is that doing this allows us to take advantage of the extra layout features of flexbox for browsers that support it while also providing a working inline-block grid for those that don't.

You don't need to use reflex as a grid. If you prefer you can simply use it as a set of vendor-prefixed helper classes and/or mixins to get your flexbox solution off the ground faster.

When using the grid it's best to add the class reflex-wrap to the container as this means you can stack columns up next to each other without worrying if they add up to 100% width. This sets flex-wrap: wrap on the reflex-container which forces it to wrap. In the inline-block fallback your grid items will wrap automatically anyway.

The use of display: flex in reflex gives us equal height divs by default in browsers that support flexbox because the default value of align-items is stretch
