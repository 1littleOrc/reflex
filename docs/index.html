Reflex

What is Reflex?

Reflex is a responsive flexbox grid with inline-block legacy support. It gives you the ability to easily take advantage of flexbox for layout with a reasonable feeling of warmth and security that things will still look ok in older browsers. Browser support for flexbox is in a pretty good state right now and is growing all the time. At the time of writing, full global flexbox support is at 81.74% and if we include partial support it is at 92.11% so I propose that it's acceptable to start taking advantage of flexbox

What is Reflex good for?

  • Dynamically generated grids of content e.g. a set of products for sale
  • Easily creating a grid where every item has the same height
  • Creating a grid where items in the grid never push each other out of the way
  • Content layout taking advantage of flexbox where supported
  • A set of vendor-prefixed helper classes and mixins to get your flexbox solution off the ground faster

It is not intended as a replacement for more fully featured css frameworks such as bootstrap and does not include css for anything else apart from the flexbox/inline-block grid. In fact, It's easy to use this in addition to your existing css framework and only use it when you need some flexbox magic or for easily creating a grid of dynamically generated, same height content such as a list of products.

Reflex grid introduction

The reflex-container class sets a section of your page up as display: flex. It's children with a class like reflex-col-6 are also set as display: flex with a fallback of display: inline-block with strict widths which allows them to act as a grid. The idea is that doing this allows us to take advantage of the extra layout features of flexbox for browsers that support it while also providing a working inline-block grid for those that don't.

By default Reflex is set up as a 12 column grid but it can be modified to support more of less columns

As a consequence of the grid container and the grid elements being set to display: flex most of the helper classes can be applied to the grid items themselves which gives you a really flexible grid structure with (for example) column based grids inside row based grids. There are some examples later on to demonstrate some of this.

Here is an example of the bare minimum markup you need to create a grid:

<div class="reflex-container">
    <div class="reflex-col-6">
        <content>
    </div>
    <div class="reflex-col-6">
        <content>
    </div>
</div>

Which gives you this:

Browser support

Browser support for flexbox is in a pretty good state right now and regardless, Reflex should fall back to an inline-block layout for most browsers that do not support flexbox explicitly. Reflex uses some css hacks to help the inline-block grid work on older browsers.

If you want to use the grid with breakpoints in ie8 and below you will need to use respond.js which enables media query support in those browsers via JavaScript.

IE7 does not support the box-sizing model which this grid uses. So if you need to support IE7 you will need to avoid using the reflex-padding-* helper classes and instead apply margin to a child element of the grid columns

Making the grid wrap

You can add the class reflex-wrap to the container which sets flex-wrap: wrap and forces it to wrap into rows.

In the inline-block fallback your grid items will wrap automatically when they add up to 100% width (12 column units by default)

<div class="reflex-container reflex-wrap">
    <div class="reflex-col-6">
        <content>
    </div>
    <div class="reflex-col-6">
        <content>
    </div>
    <div class="reflex-col-8">
        <content>
    </div>
</div>

Browser support for grid padding

IE7 does not support the box-sizing model which this grid uses. So if you absolutely need to support IE7 you will need to avoid using the reflex-padding-* helper classes and instead apply margin to a child element of the grid columns

Changing your markup to something like this will handle the lack of support for the box-sizing model with the downside of requiring more markup:

<div class="reflex-container">
    <div class="reflex-col-6">
        <div class="<classname>">
            <content>
        </div>
    </div>
    <div class="reflex-col-6">
        <div class="<classname>">
            <content>
        </div>
    </div>
</div>

Where your class is something like this:

.classname {
    padding: 1em;
    margin: 1em;
}

Which gives you the same end result but with better legacy support:

Using flex-direction: column or flex-direction: column-reverse really does not fit with standard grid usage and is probably better suited to being applied to grid items directly to force them into column layout. Here's an example:

<div class="reflex-container">
    <div class="reflex-col-4 reflex-justify-center reflex-direction-column">
        <h2>1st</h2>
        <p>I am <strong>the best</strong></p>
    </div>
</div>

Ordering

Ordering has no support in the inline-block fallback

These helper classes allow you to easily set order to your grid elements and control the rendered order. By default there are 12 ordering helpers. If you need more then they can be modified

.reflex-order-<n>

uses order: n

<div class="reflex-container">
    <div class="reflex-col-4 reflex-order-2">
        <p>I am <strong>first</strong> in the markup but ordered second</p>
    </div>
    <div class="reflex-col-4 reflex-order-3">
        <p>I am <strong>second</strong> in the markup but ordered third</p>
    </div>
    <div class="reflex-col-4 reflex-order-1">
        <p>I am <strong>third</strong> in the markup but ordered first</p>
    </div>
</div>
